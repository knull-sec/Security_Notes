# What is Authenticaion
## Difference between Authentication and Authorization
### **Authentication**

- **Definition:** The means of proving *who you are*.
- **Question:** "Who are you?"
- **Goal:** Verifying identity (e.g., logging in).

### **Authorization**

- **Definition:** Determining the permissions valid for a specific identity.
- **Question:** "What can you access?"
- **Goal:** Verifying access rights (e.g., admin vs. standard user privileges).

## **Proving Identity or To Authentcate : The Factors(way) of Authentication**

Identity is generally proven using one (or a combination) of three primary factors:

### **1. Something You Know**

- **Examples:** Username, Password, Security Questions.
- **Security Assessment:** Relying solely on this factor is considered weak security. Passwords can be leaked, guessed, or users may choose weak strings.

### **2. Something You Have**

- **Examples:** A physical device, a mobile phone, a hardware token.
- **Mechanism:** You receive a notification or code on a physical object you possess to approve the login.

### **3. Something You Are**

- **Examples:** Biometrics (Fingerprint, Retina scan, Face ID).
- **Context:** often used in high-security physical locations (e.g., data centers) or modern mobile apps.
